    3.2 Indique si cada uno de los siguientes enunciados es verdadero o falso. Si es falso, explique por qué.
        a) Cualquier tipo de input particular de formulario de HTML5 debe desplegarse de manera idéntica en todos los navegadores compatibles con HTML5. Falso. El despliegue de los tipos input puede variar de un navegador a otro.
        b) Cuando se coloca el enfoque em el campo de texto (es decir, si el cursor está en el campo de texto), se enví el texto del atributo placeholder al servidor. Falso. Cuando se coloca el enfoque en el campo de texto, el texto de palceholder desaparece. No se "envía" cuando el usuario hace clic en el botón Enviar (a menos que el usuario escriba el mismo texto).
        c) No es necesario incluir el atributo autofocus en sus formularios. Verdadero.
        d) Los nuevos tipos de input de HTML5 tienen validación automática del lado del cliente, con lo cual se elimina la necesidad de agregar secuencias de comandos (script) complicadas a sus formularios para validar la entrada del usuario y se reduce la cantidad de datos inválidos que se envín. Verdadero.
        e) El tipo input range se valida automáticamente de manera intrínseca cuando el navegador lo despliega como un control deslizante, ya que el usuario no puede mover el control fuera de los límites del valor mínimo o máximo. Verdadero.
        f) HTML5 cuenta con validadción automática para el tipo input tel. Falso. La longitud y el formato de los números telefónicos varí de manera considerable con base en la ubicación, por lo que la validación se vuelve bastante compleja, así que HTML5 no cuenta con validación automática para el tipo input tel. Para asegurar que el usuari introduzca el número en un formato apropiado, usamos el atributo pattern.
        g) Si el usuario introduce un URL con un formato inapropiado en un tipo input url, no se validará. HTML5 no valida si el URL introducido existe en realidad. Verdadero.
        h) El elemnto nav muestra un menú desplegable de hipervínculos. Falso. El elemnto nav agrupa vínculos de navegación.
        i) El elemnto header puede usuarse sólo una vez en una página. Falso. El elemento header puede usarse varias veces en una página y a menudo incluye encabezados de HTML (h1 a h6).
        j) Los elementos nav pueden anidarse en un elemento aside. Verdadero.
        k) Podemos usar el elemento brk para evitar dividir palabras en lugares incorrecto. Falso. Puede usar el elmento wbr para evitar dividir las palabras en lugares incorrectos.
